# cs-kickstart — working on this repo

This repo **is** the project-initiation starter kit, not a project generated by it.
You're editing the kit's guides, adapters, and runner — the machinery teams clone to
run the 8-step initiation process. (Step 6 generates a *downstream* CLAUDE.md for
projects that use the kit; this file is for maintaining the kit.)

## The one rule that matters: source of truth vs. adapters

Source of truth lives in `docs/guides/proj-init/`:

- `_run-step.md` — shared execution workflow every step runs
- `_steps.yml` — step registry (branch, outputs, owner, reviewer, upstream, special actions)
- `01`–`08` numbered guides — per-document rules
- `_overview.md`, `doc-status.md`, `doc-update.md` — process + utility workflows
- `.claude/roles/` — Product Owner / Solution Architect role context the runner loads

Adapters are thin and carry NO logic:

- `.claude/commands/proj-init-*.md` (Claude Code)
- `.github/prompts/proj-init-*.prompt.md` (GitHub Copilot)

Put behavior in the guide / `_run-step.md` / `_steps.yml`. Never duplicate workflow
rules into an adapter. If you're writing the same instruction into two adapters, it
belongs in the shared layer instead.

## Keep layers in sync

Changing one step usually touches up to four places — update all that apply:

1. the step guide (`docs/guides/proj-init/NN-*.md`)
2. `_steps.yml` (if metadata/outputs/order/reviewer changed)
3. `.claude/commands/proj-init-*.md`
4. `.github/prompts/proj-init-*.prompt.md`

Adding/removing/reordering a step ALSO means the `README.md` step table and `_overview.md`.

## Protected blocks — do not rewrite

`.github/copilot-instructions.md` contains an `INITIATION-RUNNER` block fenced by HTML
comment markers. Preserve it verbatim, markers included. (Step 8 strips it in downstream
projects; in this repo, leave it.)

## No build toolchain

There is no `package.json`. `src/index.ts` is a stub that exists only so semantic
indexers have a source file — don't build a Node/TS toolchain around it or treat it as
product code.

## graphify

This repo has a knowledge graph at `graphify-out/`.

- Codebase questions: run `graphify query "<question>"` first (scoped subgraph, smaller
  than GRAPH_REPORT.md or grep). Use `graphify path "<A>" "<B>"` for relationships,
  `graphify explain "<concept>"` for a focused concept.
- Broad navigation: `graphify-out/wiki/index.md` (if present). Architecture review:
  `graphify-out/GRAPH_REPORT.md`.
- After changing files, run `graphify update .` to refresh the graph (AST-only, no API cost).

The `/graphify` trigger and skill invocation are defined in `.claude/CLAUDE.md`.
